# ALKE WALLET — ENTREGA MÓDULO 2

## SITUACIÓN INICIAL
- **UNIDAD SOLICITANTE:** Equipo de desarrollo de una empresa fintech.  
- **PROBLEMA:** Crear un Front‑end dinámico para una wallet digital que permita a los usuarios gestionar activos financieros (login, agenda de contactos, transacciones y historial).

## OBJETIVO
- **OBJETIVO:** Desarrollar una vista funcional de "Alke Wallet" que permita a los usuarios: registrarse, iniciar sesión, ver saldo, depositar, retirar, agendar contactos, enviar/recibir fondos simulados y consultar historial de transacciones.

## REQUERIMIENTOS

### REQUERIMIENTOS GENERALES
- **REGISTRO E INICIO DE SESIÓN** con credenciales.  
- **VISUALIZACIÓN DE SALDO** y administración de fondos (depósito, retiro).  
- **ENVÍO Y RECEPCIÓN DE FONDOS SIMULADOS.**  
- **HISTORIAL COMPLETO DE TRANSACCIONES.**

### REQUERIMIENTOS TÉCNICOS
- **TECNOLOGÍAS A USAR:** HTML, CSS, JavaScript, Bootstrap, jQuery.  
- **INTERFAZ RESPONSIVE** y uso de componentes de Bootstrap (formularios, modales, botones).

## QUÉ SE VA A VALIDAR
- **LEGIBILIDAD Y ORGANIZACIÓN DEL CÓDIGO.**  
- **IMPLEMENTACIÓN FUNCIONAL** de los flujos solicitados: registro/login, depósito, retiro, envío, historial.  
- **EXPERIENCIA DE USUARIO** y funcionamiento responsive.

## REFERENCIAS
- **BOOTSTRAP:** https://getbootstrap.com/  
- **JQUERY:** https://jquery.com/  
- **CONTROL DE VERSIONES:** Repositorio en GitHub

## PANTALLAS Y ARCHIVOS A ENTREGAR
- `index.html` — Entrada principal  
- `login.html` — Pantalla de inicio de sesión  
- `register.html` — Pantalla de registro  
- `menu.html` — Menú principal (saldo y accesos)  
- `deposit.html` — Formulario de depósito  
- `retirar.html` — Formulario de retiro  
- `sendmoney.html` — Agenda y envío de dinero (modales y búsqueda)  
- `transactions.html` — Historial de movimientos

## PASOS BÁSICOS DE PRUEBA
1. Abrir `index.html` en el navegador.  
2. Crear cuenta en `register.html`.  
3. Iniciar sesión en `login.html`.  
4. Realizar depósito en `deposit.html`.  
5. Enviar dinero via `sendmoney.html`.  
6. Revisar historial en `transactions.html`.
